### 2.2 为什么分频？怎么分的？
**Q3: 为什么在这个项目中要设计分频器？具体是怎么分频的？**
**参考回答：**
> **1. 为什么分频**：
> FPGA 实验板上的板载晶振提供的是 **50MHz** 的系统时钟（每秒振荡 5000 万次，周期仅为 20ns），这对于人类感知的倒计时（秒，即 1Hz）来说频率实在太高了。如果直接用 50MHz 时钟去扣减时间，24小时的倒计时会在几微秒内瞬间扣减完，所以必须进行时钟分频。
>
> **2. 怎么分的（计数器分频法）**：
> 在系统时钟 `clk50m` 的每一个上升沿，让一个 26 位的计数器自增。
> * **分频公式**：计数终值 $N = \frac{f_{in}}{f_{out}} = \frac{50,000,000\text{Hz}}{1\text{Hz}} = 50,000,000$。
> 当计数器从 `0` 计数到 `49,999,999` 时（正好累加了 5000 万次系统时钟周期，耗时整整 1 秒），将计数器清零，同时输出一个系统时钟周期宽度的**高电平使能脉冲** `pulse_1hz <= 1'b1`；在其余时间保持为 `0`。
>
> **3. 为什么不直接产生一个新时钟，而是产生使能脉冲（高分加分点 🌟）**：
> 我们并没有在代码中生成一个真正的分频时钟线（例如用分频后的信号作为其他 always 块的 clk 触发源），而是产生一个**时钟使能脉冲（Clock Enable Pulse）**。在顶层模块中，我们使用全局主时钟 `clk50m`，通过 `if (pulse_1hz)` 条件来触发计时扣减。这种设计最符合工业级 FPGA 规范，能够避免多时钟域导致的“时钟抖动和时序违规（Clock Skew）”。

### 2.3 按键消抖模块原理
**Q4: 按键消抖模块（`ax_debounce.v`）的原理是什么？为什么是 20ms？**
**参考回答：**
> **1. 为什么要消抖**：
> 物理按键在按下和释放的瞬间，由于金属弹片的弹性接触会产生 5~10ms 左右的机械抖动。如果不加消抖，FPGA 极快的运行速度会把一次按键动作识别为十几次甚至几十次“快速按下”，导致按一下按键，时间就连续跳变好多次。
>
> **2. 消抖的原理**：
> 我们在代码中采用的是**两级寄存器打拍检测电平变化 + 计数器滤波延迟**：
> * 首先使用两个寄存器对输入的按键状态打拍暂存：`btn_reg0 <= btn_in;`，`btn_reg1 <= btn_reg0;`。
> * 一旦检测到两级寄存器状态不一致（`btn_reg0 != btn_reg1`），说明发生了电平跳变或抖动，此时计数器 `cnt` 立即归 0，重新开始计数。
> * 只有当按键电平连续稳定维持 1,000,000 个系统时钟周期时（在 50MHz 时钟下，计算可得 $1,000,000 \times 20\text{ns} = 20\text{ms}$），才说明按键电平已经稳定。
> * 在 20ms 延时计满时，如果按键当前稳定状态为低电平（0，代表按下），且系统此前记录的按键状态为释放，则输出一个时钟周期的有效脉冲 `btn_pulse <= 1'b1`，触发一次调整动作。

### 2.4 数码管动态扫描驱动原理
**Q5: 数码管是如何显示 HH-MM-SS 的？动态扫描为什么人眼看不见闪烁？**
**参考回答：**
> **1. 动态扫描的原理**：
> 实验板上有 8 位数码管，它们的 8 个段选（a, b, c, d, e, f, g, p）是物理并联在一起的。如果我们采用静态显示，所有数码管都会显示相同的数字。
> 动态扫描则是利用**人眼的视觉暂留效应**。在极短的时间内，轮流点亮每一位数码管，并送出对应的段选数据。只要切换速度足够快，人眼看起来就是 8 位数码管同时亮起。
>
> **2. 扫描频率与参数计算**：
> * 在 [seg7_disp.v](file:///d:/应用/qq_setup_10974/Beals/seg7_disp.v) 中，我们设计了一个 15 位的扫描计数器 `scan_cnt`，在 50MHz 下自增。
> * 我们提取其最高 3 位 `[14:12]` 作为位选切换信号 `sel`（3位宽对应 0-7，恰好选择 8 个数码管）。
> * 每当低 12 位自增溢出（即经过 $2^{12} = 4096$ 个时钟周期），`sel` 就会加 1。切换周期为 $4096 / 50\text{MHz} \approx 81.9\mu\text{s}$。
> * 扫描完一轮（8个数码管全部点亮一遍）所花的时间为 $81.9\mu\text{s} \times 8 \approx 655\mu\text{s}$，折算成扫描频率大约是 **1.5kHz**。
> * 只要扫描频率大于 50Hz，人眼就感觉不到闪烁。我们的 1.5kHz 扫描频率远远高于此要求，因此数码管显示非常稳定，毫无鬼影和闪烁感。

### 2.5 顶级控制逻辑与借位设计
**Q6: 你的顶级模块是怎么管理状态的？倒计时的借位逻辑是怎么实现的？**
**参考回答：**
> **1. 状态管理**：
> 我们在顶层定义了 `work_state` 寄存器：
> * `work_state == 1'b0`（设置状态）：此时计时停止，可以通过按键分别调节小时（`hh`）、分钟（`mm1`, `mm0`）、秒数（`ss1`, `ss0`）。
> * `work_state == 1'b1`（倒计时状态）：当设置好时间后，按下 `sw_ok` 启动倒计时。
>
> **2. 借位逻辑**：
> 我们采用的是 **BCD 码（十进制）直接扣减**的借位方式，而不是把总时间化为总秒数再递减（避免了使用复杂的除法器，极大节省了 FPGA 的查找表资源）：
> * 当 1Hz 脉冲 `pulse_1hz` 有效时，如果秒个位 `ss0` 大于 0，则 `ss0 <= ss0 - 1`。
> * 如果 `ss0` 等于 0，且秒十位 `ss1` 大于 0，则将 `ss0` 重置为 `9`，并将 `ss1 <= ss1 - 1`（向十位借位）。
> * 依此类推，秒十位重置为 `5`，分个位递减；分个位重置为 `9`，分十位递减；分十位重置为 `5`，小时 `hh` 递减。
> * 当所有计时位都为 0 时（`hh == 0 && mm1 == 0 && mm0 == 0 && ss1 == 0 && ss0 == 0`），倒计时结束，跳转到报警状态（`alarm <= 1'b1`）。

## ⚡ 第三部分：硬件特性与电平逻辑（硬核考点）

### Q7: 你们用的实验板上，数码管是共阴极还是共阳极？段选和位选分别是什么电平有效？
**参考回答：**
> “根据我们的代码和实验板原理图设计：
> 1. **段选信号**：是**高电平有效**（1点亮，0熄灭）。例如数字 `0` 的段码为 `8'b00111111`，对应的 g~a 段中，a,b,c,d,e,f 为高电平 `1` 点亮，g 熄灭。
> 2. **位选信号**：在 FPGA 输出端也是**高电平有效**（`com[sel] = 1'b1` 时选中）。
> 3. **极性与共阴共阳判断**：由于段选和位选在 FPGA 输出侧都是高电平有效，而实际上我们的数码管是**共阴极**的。
>    * **原因在于硬件驱动电路**：实验板上的数码管位选引脚并没有直接连在 FPGA 芯片管脚上，而是接在 **NPN 三极管** 或 **反相驱动芯片** 上。
>    * 当 FPGA 位选管脚输出高电平 `1` 时，会导通 NPN 三极管（或经过反相器将输出拉低为 `0`），从而把数码管的公共阴极端拉低接地。因此，它实质上是一个共阴极数码管。这也展现了数电设计中‘软硬件电平配合’的典型应用。”

### Q8: 实验板上的复位按键和功能按键是高电平有效还是低电平有效？
**参考回答：**
> “按键在常态下接上拉电阻，保持为高电平 `1`。当按键被按下时，开关闭合将电平拉低到 `0`。因此，按键是**低电平有效**的。
> 在代码中，按键消抖模块检测的是输入信号从 `1` 变到 `0` 的**下降沿**，并进行 20ms 的电平滤波确认。”

## 🎯 第四部分：答辩现场高频追问 FAQ（临场通关）

### Q9: 如果老师让你把这个倒计时改成“正向计时器（秒表）”，你该怎么改代码？
**参考回答：**
> “改动非常简单，只需要修改顶层模块 `top_countdown.v` 中 `work_state == 1'b1` 时的计数逻辑：
> 1. 原本的逻辑是递减（`- 1`）并在全零时借位；
> 2. 我们只需将其修改为**递增（`+ 1`）并在达到极限时向高位进位**。
>    * 例如：在 `pulse_1hz` 有效时，`ss0 <= ss0 + 1`；
>    * 当 `ss0 == 9` 时，清零 `ss0 <= 0`，同时让秒十位 `ss1 <= ss1 + 1`；
>    * 当 `ss1 == 5 && ss0 == 9` 时，清零 `ss1 <= 0, ss0 <= 0`，并让分钟个位递增，以此类推。
> 3. 同时把报警触发条件修改为‘计时达到预设目标时间’，即可变为正向计时器。”

### Q10: 如果老师要你增加一个“暂停/继续”功能，你怎么改？
**参考回答：**
> “我们可以通过一个按键（比如复用 `sw_ok` 或者新增一个按键）来控制一个暂停标志寄存器 `pause_flag`：
> 1. 定义一个 `reg pause_flag;`，在每次按下暂停键时进行翻转：`pause_flag <= ~pause_flag;`。
> 2. 在顶层模块的时钟扣减判断中，在 `else if(pulse_1hz)` 条件下，增加暂停标志的判断：
>    ```verilog
>    else if(pulse_1hz && !pause_flag) begin
>        // 执行时间扣减逻辑...
>    end
>    ```
> 3. 这样，当 `pause_flag` 为 1 时，即使 1Hz 脉冲到来，系统也会屏蔽它，时间保持不变，再次按下则继续计时。”

### Q11: 在调试中你遇到过什么困难？怎么解决的？
**参考回答：**
> “我们在调试中遇到了数码管显示‘鬼影’的问题。
> * **问题表现**：虽然数码管显示是正确的，但是总能隐约看到相邻数码管的残影。
> * **问题分析**：这是由于动态扫描切换位选时，上一位的段选数据还没有来得及彻底关闭，下一位的位选就已经打开了，导致短暂的错乱显示。
> * **解决方案**：在动态扫描逻辑中，我们在每次切换 `sel` 的瞬间，短暂将段选置为全熄灭状态（即‘消隐’），之后再输出新的段选段码；或者采用稍微高一点的扫描计数器溢出值，给数码管留出充足的熄灭反应时间。我们在模块中对输入数字不是 0-9 时赋 `8'b00000000`（全熄灭），有效地避免了这个问题。”

### Q12: 为什么要使用 FPGA（Verilog）来做倒计时，而不是用单片机（MCU）？
**参考回答：**
> “这是因为 FPGA 和单片机有本质的区别：
> 1. **执行方式不同**：单片机是 **CPU 架构**，采用**串行顺序执行**。如果想要同时处理数码管扫描、按键消抖和计时，需要写定时器中断或者在主循环里不停轮询，极易发生延迟和互相干扰。而 FPGA 是**硬件级并行执行**。数码管动态扫描、按键滤波和时间计算是完全独立的硬件电路，在同一个时钟沿同时工作，没有任何互锁和延迟，时序控制极其精准。
> 2. **开发本质不同**：单片机开发是**写软件代码**让 CPU 执行；FPGA 开发是使用 Verilog **描述硬件电路**。我们写出的 Verilog 代码最终会被 Quartus 综合成一个个真正的查找表（LUT）、触发器（FF）等物理逻辑门电路。这也是我们选择 FPGA 来进行设计的核心价值所在。”
